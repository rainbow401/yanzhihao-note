SpringBoot必会知识

SpringBoot简介
  • Spring 应用需要进行大量的配置，各种 XML 配置和注解配置让人眼花缭乱，且极容易出错。为了简化 Spring 应用的搭建和开发过程，Pivotal 团队在 Spring 基础上提供了一套全新的开源的框架，它就是 Spring Boot。
  • Spring Boot 具有 Spring 一切优秀特性，Spring 能做的事，Spring Boot 都可以做，而且使用更加简单，功能更加丰富，性能更加稳定而健壮。
  • 优点：
    1. 可以以jar包的形式独立运行。"java -jar xx.jar"
    2. 提供了starter特性，简化Maven配置。Spring Boot官方提供了大量starter，例如：spring-boot-starter-redis、spring-boot-starter-data-mongodb 和 spring-boot-starter-data-elasticsearch 等
    3. 提供了大量的自动配置来简化项目的开发
    4. 不再需要任何xml配置即可实现Spring的所有配置
Spring Boot的配置文件
  • 配置文件类型：
  1. Properties文件：
    • 是java中很常用的一种配置文件，文件后缀为“.properties”，属文本文件，文件的内容格式是“键=值”的格式，用“#”作为注释。
    • Properties类: java中提供的配置文件的操作类Properties类（java.util.Properties）
    • 演示：org.example.properties
  2. YAML文件：
    • 全称 YAML Ain't Markup Language，它是一种以数据为中心的标记语言。
    • 以 .yml 或 .yaml 结尾
    • 解析需要类库：snakeyaml
    • 语法：
    1. 使用缩进表示层级关系。
    2. 缩进时不允许使用 Tab 键，只允许使用空格。
    3. 缩进的空格数不重要，但同级元素必须左侧对齐。
    4. 大小写敏感。
    5. 用“#”作为注释。
    • 演示：org.example.yml

  • 配置绑定：SpringBoot启动，在把类装入Spring容器时同时把配置文件中的值与类中对应的属性进行绑定。

    1. 使用 @ConfigurationProperties 注解
      • 通过 Spring Boot 提供的 @ConfigurationProperties 注解，可以将全局配置文件中的配置数据绑定到类的实例
      • 注意事项：
      1. 如果我们想要使用 @ConfigurationProperties 注解进行配置绑定，那么首先就要保证该对象在 Spring 容器中，所以需要用到 @Component 注解来添加组件到容器中。
      2. 注解 @ConfigurationProperties(prefix = "person") ，它表示将这个 JavaBean 中的所有属性与配置文件中以“person”为前缀的配置进行绑定。
      3. 支持松散绑定
      • 演示: org.example.cp
    2. 使用 @Value 注解
      • 当我们只需要读取配置文件中的某一个配置时，可以通过 @Value 注解获取。
      • 注意事项：
      1. 如果我们想要使用 @Value 注解进行配置绑定，那么首先就要保证该对象在 Spring 容器中，所以需要用到 @Component 注解来添加组件到容器中。
      2. 被注解@Value修饰的属性，在全局配置文件中必须有与之对应的配置值，否则会在SpringBoot启动装载该实例的时候抛异常
      3. 默认值使用： @Value("${test.value:default}")
      • 演示: org.example.v
    3. @Value 与 @ConfigurationProperties 对比
    区别 @ConfigurationProperties @Value
    注解标注位置不同 标注在类(class)上 标注在字段(field)上
    功能不同 用于批量绑定配置文件中的配置 只能一个一个的指定需要绑定的配置
    松散绑定支持不同 支持松散绑定 不支持松散绑定
    SpEL 支持不同 不支持 SpEL 表达式 支持 SpEL 表达式
    复杂类型绑定支持不同 支持所有类型数据的绑定，例如 Map、List、Set、以及自定义类的对象等 默认只支持基本数据类型的封装，例如字符串、布尔值、整数等类型，可通过SpEL进行扩展
    应用场景不同 适合复杂的配置类的实例映射 适合单配置项的值映射

    1. @PropertySource:

      • 如果将所有的配置都集中到 application.properties 或 application.yml 中，那么这个配置文件会十分的臃肿且难以维护，因此我们通常会将与 Spring Boot 无关的配置（例如自定义配置）提取出来，写在一个单独的配置文件中，并在对应的类上使用 @PropertySource 注解指向该配置文件。
      • 演示: org.example.cp

    2. @EnableConfigurationProperties:

      • 表示对 @ConfigurationProperties 的内嵌支持，默认会将对应Properties Class作为bean注入的IOC容器中，即在相应的Properties类上不用加 @Component注解

    3. @Conditional：

      • 判断当满足某种条件后才初始化一个bean或者启用某些配置
      • 用在由 @Component、 @Configuration等注解标记的类上面或者由@Bean标记的方法上。如果一个 @Configuration类标记了 @Conditional，则该类中所有标识了 @Bean的方法和 @Import注解导入的相关类将遵从这些条件。
      • 需要实现Condition接口提供一个判断条件的实现。

  • 默认配置文件：

    • 最常见的情况，Spring Boot 在启动时会将 resources 目录下的 application.properties 或 apllication.yml 作为其默认配置文件
    • Spring Boot 启动时会扫描以下 5 个位置的 application.properties 或 apllication.yml 文件，并将它们作为 Spring boot 的默认配置文件：
    1. file:/config/*/
2. file:/config/
3. file:/
4. classpath:/config/
5. classpath:/
    • 注意：file: 指当前项目根目录(与pom.xml同级)；classpath: 指当前项目的类路径，即 resources 目录。
    • 以上所列五个文件路径的配置文件都会被加载，且它们优先级依次降低，序号越小优先级越高。其次，位于相同位置的 application.properties 的优先级高于 application.yml。
    • 所有位置的配置文件都会被加载，高优先级配置会覆盖低优先级配置，形成互补配置：
      1. 存在相同的配置项时，高优先级的配置项会覆盖低优先级的配置项；
      2. 存在不同的配置项时，高优先级和低优先级的配置项取并集。
    • 演示： org.example.co

  • 外部配置文件：

    • 除了默认配置文件，Spring Boot 还可以加载一些位于项目外部的配置文件。我们可以通过如下 2 个参数，指定外部配置文件的路径：
    1. spring.config.location
      • 使用方式： java -jar {JAR} --spring.config.location={外部配置文件全路径}
      • 使用该参数指定配置文件后，会使项目默认配置文件（application.properties 或 application.yml ）失效，Spring Boot 将只加载指定的外部配置文件。
    2. spring.config.additional-location
      • 使用方式： java -jar {JAR} --spring.config.additional-location={外部配置文件全路径}
      • 与 --spring.config.location 不同，--spring.config.additional-location 不会使项目默认的配置文件失效，使用该命令行参数添加的外部配置文件会与项目默认的配置文件共同生效，形成互补配置，且其优先级是最高的，比所有默认配置文件的优先级都高。

  • 配置加载顺序：

    • Spring Boot 不仅可以通过配置文件进行配置，还可以通过环境变量、命令行参数等多种形式进行配置。这些配置都可以让开发人员在不修改任何代码的前提下，直接将一套 Spring Boot 应用程序在不同的环境中运行。
    • 以下是常用的 Spring Boot 配置形式及其加载顺序（优先级由高到低）：
    1. 命令行参数: java -jar xxx.jar --{参数1}={参数值1} --{参数2}={参数值2}
    2. 来自 java:comp/env 的 JNDI 属性
    3. Java 系统属性（System.getProperties()） java -jar -D{参数1}={参数值1} -D{参数2}={参数值2} xxx.jar
    4. 操作系统环境变量
    5. RandomValuePropertySource 配置的 random.* 属性值
    6. 配置文件（YAML 文件、Properties 文件）
    7. @Configuration 注解类上的 @PropertySource 指定的配置文件
    8. 通过 SpringApplication.setDefaultProperties 指定的默认属性
    • 以上所有形式的配置都会被加载，当存在相同配置内容时，高优先级的配置会覆盖低优先级的配置；存在不同的配置内容时，高优先级和低优先级的配置内容取并集，共同生效，形成互补配置。
    • 演示： org.example.co

Spring Boot Starter

区别于Spring项目创建后的繁琐的手工配置，Spring Boot Starter机制使Spring Boot项目在创建完成后，无需任何配置或者修改代码，即可直接运行。

  • 什么是Starter:

    • 本质：一个基于Maven的类库
    • 包含内容：某应用场景中所需要使用的一系列的其他组件以及相应的工具类，在被项目引用后会在项目启动时会自动装入Spring容器。
    • 使用方法：通过Maven引入。
    <dependency>
	<groupId>org.springframework.boot</groupId>
	<artifactId>spring-boot-starter</artifactId>
    <version>1.0.0</version>
</dependency>

  • Starter机制的组成

    • Maven的依赖传递特性
      • 查看某maven组件的依赖，使用插件：mvn dependency:tree
      • Maven依赖标识符的组成：{groupId}:{artifactId}:{packaging}:{version}:{scope}
    • Spring boot的自动配置

  • Spring Boot自动配置原理

    • 典型的Spring Boot应用的启动类一般均位于 src/main/java根路径下，比如 DemoApplication类：
    @SpringBootApplication
public class DemoApplication {

    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}
    • SpringApplication.run则负责启动引导应用程序。
    • @SpringBootApplication开启组件扫描和自动配置，@SpringBootApplication是一个复合 Annotation，它将三个有用的注解组合在一起：
    @Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Inherited
@SpringBootConfiguration
@EnableAutoConfiguration
@ComponentScan(excludeFilters = {
        @Filter(type = FilterType.CUSTOM, classes = TypeExcludeFilter.class),
        @Filter(type = FilterType.CUSTOM, classes = AutoConfigurationExcludeFilter.class) })
public @interface SpringBootApplication {
    // ......
}
    1. @SpringBootConfiguration就是@Configuration，它是Spring框架的注解，标明该类是一个JavaConfig配置类
    2. @ComponentScan启用组件扫描
    3. @EnableAutoConfiguration注解表示开启Spring Boot自动配置功能，Spring Boot会根据应用的依赖、自定义的bean、classpath下有没有某个类 等等因素来猜测你需要的bean，然后注册到IOC容器中
